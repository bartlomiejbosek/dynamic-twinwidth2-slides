`\(M\) is \(d\)-twin-ordered iff it admits such a contraction sequence.`

`Throughout the paper, the order is fixed.`

5. **a) Tytuł**
**Main theorem**

**b) Cele slajdu**
- Pokazać główny wynik wcześnie i bardzo wyraźnie.
- Ustawić wszystko, co dalej będzie wyjaśniane jako proof idea tego jednego twierdzenia.
- Uczciwie zaznaczyć różnicę między pamięcią dynamiczną a statycznym wynikiem bitowym.

**c) Treść slajdu**
- Centralny theorem box:
\[
\textbf{Theorem 1.}
\]
- `Let \(d\in\mathbb N\) and \(M\in\{0,1\}^{n\times n}\) be updated over time, always remaining \(d\)-twin-ordered.`
- `There is a dynamic data structure using \(O_d(n)\) memory and supporting:`
- `Init(n,\mathcal K)` in `\(O_d((n+|\mathcal K|)\log\log n)\)`
- `QueryBit(i,j)` in `\(O(\log\log n)\)` expected worst-case time
- `Update(i,j)` in `\(O(\log\log n)\)` expected worst-case time
- Dwa krótkie dopiski pod boxem:
- `\(\mathcal K\)` is a list of disjoint rectangles representing all ones
- `word RAM: \(O_d(n)\) memory means \(O_d(n\log n)\) bits`
- Jedna mała linia na dole:
`Only initialization and memory depend on \(d\).`

**d) Bardzo precyzyjny opis rysunku**
- Bez osobnej figury narracyjnej; głównym obiektem ma być theorem box.
- Pod theorem boxem mała tabelka `static vs dynamic` z trzema wierszami: `space`, `query`, `update`.
- W wierszu `space` wpisać dokładnie `static: \(O_d(n)\) bits`, `dynamic: \(O_d(n)\) memory`.
- W wierszu `query` wpisać dokładnie `static: \(O_d(\log\log n)\)`, `dynamic: \(O(\log\log n)\)`.
- W wierszu `update` wpisać dokładnie `static: —`, `dynamic: \(O(\log\log n)\)`.

**e) Zakotwiczenie źródłowe**
- Przede wszystkim: oba źródła.

7. **a) Tytuł**
**Canonical slabs: the normalization step**

**b) Cele slajdu**
- Wyjaśnić, że „jakaś” slab decomposition nie wystarcza do dynamicznej konstrukcji.
- Wprowadzić canonical slab decomposition jako właściwy obiekt utrzymywany po przebudowie.
- Pokazać, że kanoniczność nie niszczy liniowego rozmiaru.

**c) Treść slajdu**
- Trzy definicje:
- `an \(i\)-strip is a maximal all-1 slab of width 1 inside column \(i\)`
- `\(\mathtt{strips}_i\)` is the set of all \(i\)-strips
- `siblings are strips with the same row interval persisting through consecutive columns`
- Wyróżniona linia:
`A canonical slab is the union of one sibling class, and each strip belongs to exactly one canonical slab.`
- Potem:
\[
|\mathcal R| \le 4f_d(n+2)+4n
\]
- Krótki komentarz:
`Canonical slabs are still only \(O_d(n)\).`
- Drobny dopisek przy formule lub pod nią:
`Here \(f_d\) is the constant from the corner bound behind the \(O_d(n)\) estimate.`

**c) Treść slajdu**

B_i := \mathtt{strips}_i \setminus \mathtt{strips}_{i+1}
\]

- Na dole jedna linia:
`This is the first half of \(\mathtt{Decompose}(n,\mathcal K)\).`

- Dwa duże boksy:
- `\(\mathbb P\)`: static 2D orthogonal point location on the canonical slab decomposition stored at the last rebuild; in the running example it stores exactly the five slabs of `\(\mathcal P\)`
- `\(\mathbb Q\)`: hash table of recent updates since that rebuild
- Ustalona na resztę referatu zawartość bufora:
- `first \(u_1=(4,2):1\to 0\), then \(u_2=(1,2):0\to 1\)`
- `equivalently, \(\mathbb Q[(4,2)]=0\) and \(\mathbb Q[(1,2)]=1\)`
- Próg bufora:
\[
|\mathbb Q| \le 8f_d(n+2)
\]
- Krótki komentarz do progu:
`Here \(f_d\) is the same constant coming from the corner / canonical-slab bound.`
- Jedna linia intuicji:
`Current matrix = base representation in \(\mathbb P\) overlaid with the explicit overrides stored in \(\mathbb Q\).`
- W running example role obu updates są już ustalone:
- `\((4,2)\)` is a `1\to 0` flip inside the old slab `\((2,4,2,3)\)`
- `\((1,2)\)` is a `0\to 1` flip outside all slabs of `\(\mathcal P\)`
- Inwariant:
\[
(i,j)\in \mathbb Q \Rightarrow M[i,j]=\mathbb Q[i,j]
\]
\[
(i,j)\notin \mathbb Q \Rightarrow M[i,j]=1 \iff (i,j)\text{ lies in a base slab stored in }\mathbb P
\]
- Dwa krótkie punkty:
- `QueryBit`: check `\(\mathbb Q\)` first, otherwise issue a point-location query to `\(\mathbb P\)`
- `Update`: toggle the value in `\(\mathbb Q\)`; rebuild only when the threshold is reached
